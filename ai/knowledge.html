🎴 AI知识卡片

技术术语通俗解读 · 点击卡片翻转查看详解

🤖
LLM
Large Language Model

大语言模型。像ChatGPT这样的AI，能理解和生成人类语言。把它想象成一个读过亿万本书的"超级阅读者"。

🎯
Fine-tuning
微调

像给AI"上专业课"，在通用能力基础上，针对特定领域（如医疗、法律）进行额外训练。

💬
Prompt
提示词

给AI的"指令"。如何提问才能得到想要的答案？Prompt工程就是设计精准指令的艺术。

📄
Context
上下文窗口

AI能"记住"的对话长度。Context Window越大，能参考的信息越多，像手机的RAM容量。

🧠
Transformer
变换器架构

AI模型的核心"大脑结构"。像吃火锅时，你能同时关注锅里的肉、调料碗、朋友的聊天，Transformer就这样"理解"文本。

👁️
Attention
注意力机制

让AI知道"重点看哪里"的机制。就像你读句子时会注意到关键词，Attention找到最重要的部分。

🔄
Embedding
嵌入向量

把文字变成"数字坐标"。"猫"和"狗"在向量空间里距离很近，"猫"和"汽车"距离很远。让计算机理解词语关系。

🏷️
Tokenizer
分词器

把句子切成"小块"。不是按字切，而是按语义切。"火锅店"可能被切成["火锅", "店"]，而不是["火", "锅", "店"]。

⚡
RAG
检索增强生成

AI先去"查资料"再回答问题。像考试时允许翻书，答案更准确、更有依据。

📊
Quantization
量化

把模型"瘦身"。从32位精度降到8位甚至4位，让大模型能在普通电脑上运行。

🔥
LoRA
低秩适配

高效的微调方法。不改动整个模型，只添加一小块"补丁"。像给手机换壳而不是重装系统。

💾
KV Cache
KV缓存

推理时把已计算的Key-Value缓存下来，避免重复计算。像"抄作业"，但合法。

🤖
Agent
智能代理

不只是聊天，还能"做事"的AI。能搜索、订票、写代码。像火锅店的智能服务员，能看菜单、点菜、上菜。

🔄
Inference
推理

AI模型"运行"的过程。训练是学习，推理是应用。像学生考试时运用学到的知识解题。

🔧
Function Calling
函数调用

让AI调用外部函数/工具的能力。AI说"我要查天气"，实际调用天气API获取数据。

🔌
MCP
模型上下文协议

Anthropic推出的工具调用标准。让AI能统一调用各种外部工具，像"万能插座"。

📝
CoT
思维链

Chain of Thought。让AI"一步步思考"，而不是直接给答案。先推理再回答，准确率更高。

⚠️
Hallucination
幻觉

AI"胡说八道"的现象。推荐不存在的火锅店、编造虚假信息。是LLM的固有缺陷。

🎯
Temperature
温度参数

控制AI回答的"创造性"。Temperature=0保守稳定，Temperature=1天马行空。

🧠
MoE
混合专家

Mixture of Experts。大模型分成多个"小专家"，每次只激活几个。既强大又高效。
